----------------------------

   resolveBoard() → returns DEFAULT_BOARD_ID for null, non-string, empty, or unsupported
   ✓ Defensive, graceful fallback.

2. Missing board config at YAML generation:
   resolveBoardConfig() → falls back to default board, then to FALLBACK constant
   ✓ Never returns undefined; always has a valid config.

3. Board without RGB LED:
   updateLEDCompatibility() → hides rgb-led-controls, disables inputs, sets aria-hidden
   ✓ Clean UI adaptation.

4. Unsupported board in validation:
   validateConfig() → adds error with helpful message listing all supported boards
   ✓ Good user feedback.

5. Legacy code path compatibility:
   generateHardwareConfig() → returns deps.hardwareConfig for default board (backward compat)
   generateFullYAML() → if no board deps available, returns null boardConfig and falls back to hardwareConfig
   ✓ Existing behavior preserved when board feature not engaged.
